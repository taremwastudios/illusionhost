IllusionHost Documentation

Use this guide to connect your app to your IllusionHost subdomain. It explains what you must submit to us, what we provide back for verification, and platform-specific connection notes.

1) Standard Onboarding Flow

  1. You preorder and receive your order ID.
  2. We confirm your request in admin dashboard.
  3. You send us app hosting details (below).
  4. We map DNS + verify target platform.
  5. Your status changes to live.

2) What You Must Provide to Us

Required

  • Your Order ID
  • Your desired subdomain (e.g. myapp.illusionhost.net)
  • Your platform name (Vercel, Netlify, Railway, etc.)
  • Target app URL / domain generated by your platform

For DNS Verification

  • Any required TXT verification value from your platform
  • CNAME/A target value from your platform
  • Preferred root path and redirect behavior

3) What We Give Back to You

  • DNS records we configured for your subdomain
  • Verification completion confirmation
  • Live status + SSL/HTTPS availability confirmation
  • Any troubleshooting notes if target platform rejects verification

4) Platform Verification Cheat Sheet

Platform You Provide We Configure Verification Result
Vercel Project domain target + any TXT value CNAME/TXT as required Domain marked valid in Vercel
Netlify Custom domain target and verification prompts CNAME/TXT records Domain verified in Netlify dashboard
Render Service domain endpoint CNAME mapping Render custom domain active
Railway Railway-generated domain target CNAME and optional TXT Railway domain attached
Koyeb App domain + required verification CNAME/TXT records Koyeb verification complete
Cloudflare Pages Pages project domain requirements CNAME/TXT records Pages domain active

5) Typical DNS Records You May See

  • CNAME: subdomain -> cname.target.host
  • A: subdomain -> 1.2.3.4 (if platform requires IP target)
  • TXT: verification tokens (platform ownership proof)

Note: exact target values differ by platform and project. Always send us what your hosting provider shows.
